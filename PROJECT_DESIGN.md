## 一、项目定位

**DramaPipeline**（影视流水线）是一款**工业级短剧生成工具**，面向**专业级用户**和**企业用户**。核心特点：

- **非全自动**：每个流水线步骤需用户确认输出内容后，再手动分发到下一阶段
- **流水线式**：角色不同步工作，按顺序执行，用户主动触发下一步
- **控制台风格**：Web 平台采用控制台式 UI，偏专业、高效、可观测

---

| 企业用户   | MCN 机构、影视公司、内容团队 |

**核心诉求**：可控的流水线、人工审核节点、高并发任务、生成完成提醒。

## 三、流水线角色定义（10 个 Agent）

参考「群机器人」界面，影视生成流水线包含以下角色，**按顺序执行**，每步需用户确认后手动分发：

| 序号 | 角色名称           | 英文标识              | 职责简述                     |
| ---- | ------------------ | ---------------------- | ---------------------------- |
| 1    | 剧本分析机器人     | script_analysis        | 解析文本剧本，提取角色、场景、动作 |
| 2    | AI 导演团队        | ai_director            | 编排中枢，协调各环节，管理整体流程 |
| 3    | 素材分析助手       | material_analysis      | 分析原始素材，打标、兼容性检查 |
| 4    | 分镜设计助手       | storyboard_design      | 将剧本转为可视化分镜           |
| 5    | 首帧生成助手       | first_frame            | 生成场景关键帧 / 视觉参考      |
| 6    | 视频生成助手       | video_generation       | 核心视频渲染 / 生成引擎        |
| 7    | 后期合成助手       | post_production        | 视觉与音频的最终合成           |
| 8    | 角色创建大师       | character_creation     | 角色设计与一致性维护           |
| 9    | 场景生成大师       | scene_generation       | 背景与环境生成                |
| 10   | 审片助手           | review_assistant       | 质量审核与成片检查             |

**说明**：实际执行顺序可根据业务配置调整，上述为参考顺序。

## 四、核心流程

```
用户输入剧本 → 剧本分析 → [用户确认] → 分镜设计 → [用户确认] → 首帧生成 → ...
    → 视频生成 → [用户确认] → 后期合成 → [用户确认] → 审片 → [用户确认] → 完成
```

**关键原则**：

1. **每步输出后暂停**：等待用户查看、编辑、确认
2. **手动分发**：用户点击「分发到下一阶段」后，才进入下一角色
3. **可回退 / 重做**：支持在某一步重新生成或回退到上一步
4. **并发**：支持多项目 / 多任务并行，互不阻塞

## 五、Web 平台 UI 风格：控制台

- **布局**：左侧固定导航 + 右侧主内容区（参考 zeroclaw Layout）
- **配色**：深色主题，偏专业、工业感（可沿用 zeroclaw 的 electric blue 或调整为影视行业常用色）
- **导航结构**：
  - 工作台 / 总览
  - 流水线 / 任务列表
  - 任务详情（含步骤确认、分发）
  - 角色管理 / 配置
  - 通知中心（生成完成提醒）
  - 系统设置
- **交互**：步骤卡片、确认按钮、分发按钮、进度条、状态标签

## 六、技术架构（建议）

| 层级 | 技术选型 | 说明 |
| ---- | -------- | ---- |
| 前端 | Vue 3 + Element Plus + Tailwind | 或参考 zeroclaw 用 React + Tailwind 做控制台 |
| 后端 | Go + Gin | 高并发 API、任务队列、持久化 |
| 编排 | Python + FastAPI + LangGraph | 流水线编排、状态机 |
| 存储 | SQLite / PostgreSQL | 任务、步骤、用户、项目 |
| 通知 | WebSocket / SSE / 轮询 | 生成完成提醒 |

## 七、非功能需求

| 需求         | 描述                                   |
| ------------ | -------------------------------------- |

- **zeroclaw**：Agent 控制台，Dashboard、Tools、Cron、Integrations、Memory、Config、Cost、Logs、Doctor 等

  - 替换导航为「流水线 / 任务 / 通知 / 配置」
  - 新增「流水线步骤确认」「手动分发」等业务页面
  - 后端对接 Go API，而非 zeroclaw 的 Rust 服务
