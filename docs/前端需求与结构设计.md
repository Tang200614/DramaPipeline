## 一、需求归纳

### 1.1 核心能力

| 能力 | 描述 | 对应角色/模块 |
|------|------|---------------|
| 多剧本并行 | 支持多脚本同时运行，用户可切换不同剧本项目 | 项目/剧本选择器 |
| 剧本输入 | 用户上传剧本 / 用户给大纲（题目）→ 模型流水线分析生成剧本 | 剧本分析机器人 |
| 素材输入 | 用户上传 zip、rar、图片 | 素材分析助手 |
| 自定义角色 | 用户定义角色 → 模型根据角色修改剧本 | 角色创建大师 |
| 剧本切片 | 剧本切片后交给分镜设计 | 分镜设计助手 |
| 用户配置 | 剧本量、拼接字数、匹配章节 | 配置面板 |
| 知识库 | 用户上传的剧本作为当前项目知识库 | 项目级知识库 |

### 1.2 与 10 角色流水线的映射

| 角色 | 职责 | 输入来源 | 输出去向 |
|------|------|----------|----------|
| 剧本分析机器人 | 解析/生成剧本、流水线分析 | 上传剧本 / 大纲 / 知识库 | 分镜设计、角色创建 |
| AI 导演团队 | 编排中枢 | 各环节输出 | 协调分发 |
| 素材分析助手 | 分析素材包、打标 | zip/rar/图片 | 首帧、视频生成 |
| 分镜设计助手 | 剧本→分镜 | 切片后剧本 | 首帧生成 |
| 首帧生成助手 | 关键帧/视觉参考 | 分镜、素材 | 视频生成 |
| 视频生成助手 | 视频渲染 | 首帧、素材 | 后期合成 |
| 后期合成助手 | 视音频合成 | 视频、音频 | 审片 |
| 角色创建大师 | 角色设计、剧本修改 | 用户自定义角色 | 剧本分析、分镜 |
| 场景生成大师 | 背景与环境 | 分镜、素材 | 首帧、视频 |
| 审片助手 | 质量审核 | 成片 | 完成/回退 |

## 二、前端结构设计（按草图定稿）

### 2.1 三块式布局所在位置

三块式布局出现在 **流水线页面**（`/pipelines`）中，作为该页面的主内容区。

```
┌─────────────────────────────────────────────────────────────────────────┐
│ Sidebar │  Header                                                         │
│ 工作台  ├────────────────────────────────────────────────────────────────┤
│ AI 对话 │  /pipelines 流水线页面                                          │
│ 流水线  │  ┌─────────────────────────────────────────────────────────┐   │
│ 任务列表│  │  剧本选择 (Script Selection)              ← 顶部横条     │   │
│ 系统设置│  ├────────────────────────────┬────────────────────────────┤   │
│         │  │  角色流转 (左侧)            │  主内容区 (右侧)             │   │
│         │  │  每个剧本有：               │  左侧选择角色 → 右侧显示     │   │
│         │  │  • 自己的角色组             │  该角色的会话、处理进度       │   │
│         │  │  • 自己的素材库             │  确认/分发                   │   │
│         │  │  10 角色列表                │                             │   │
│         │  └────────────────────────────┴────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────────┘
```

**设计要点：**

- **流水线页面**：Sidebar 选中「流水线」时，主内容区渲染三块式布局
- **顶部**：剧本选择器，全宽，支持多剧本切换、新建、上传、大纲生成
- **左侧**：当前剧本下的角色列表（10 个 Agent），每个剧本独立一套角色、素材、数据
- **右侧**：选中角色的会话与处理进度，可回溯上一角色的输出，支持确认、分发

### 2.3 顶部：剧本选择

- **展示**：下拉选择 / 标签页切换当前剧本
- **操作**：新建剧本、上传剧本、输入大纲生成剧本
- **状态**：当前选中剧本高亮，可显示运行中/待确认等 badge

### 2.4 左侧：角色流转

- **展示**：10 角色列表，头像/图标 + 名称 + 副标题（参考专家库截图）
- **数据**：每个剧本独立，切换剧本后左侧角色列表对应更新
- **交互**：点击角色 → 右侧展示该角色的会话与进度
- **状态**：待执行 / 执行中 / 待确认 / 已完成，用颜色或图标区分

### 2.5 右侧：角色会话与进度

- **会话**：与该角色的 Chat 历史，可查看上一角色输出、当前角色回复
- **进度**：当前步骤的处理进度（进度条、状态文案）
- **操作**：确认、分发到下一阶段、回退、重做
- **输出预览**：文本/图片/视频等，按角色类型展示

### 2.6 其他页面（Sidebar 导航）

| 路由 | 页面 | 说明 |
|------|------|------|
| `/` | 工作台 | 多剧本总览、运行中任务、快捷新建 |
| `/chat` | AI 对话 | 通用 AI 对话（可后续关联剧本上下文） |
| `/pipelines` | **流水线** | **三块式布局所在页**：剧本选择 + 角色流转 + 角色会话 |
| `/tasks` | 任务列表 | 跨剧本任务、状态筛选 |
| `/config` | 系统设置 | 剧本量、拼接字数、匹配章节等默认配置 |

## 三、样式与组件规范

### 3.1 视觉风格（沿用 zeroclaw）

- **主题**：深色/亮色可切换，CSS 变量 `--dp-*`
- **卡片**：`glass-card` 毛玻璃效果
- **主色**：electric blue `#0080ff`，渐变 `#0080ff → #00d4ff`
- **字体**：系统无衬线，标题加粗，正文 14px

### 3.2 三块式布局尺寸建议

| 区域 | 宽度 | 说明 |
|------|------|------|
| 顶部剧本选择 | 100% | 高度约 48–56px，固定 |
| 左侧角色列表 | 240–280px | 可折叠，最小约 200px |
| 右侧主内容 | flex-1 | 自适应剩余宽度 |

### 3.3 组件清单

| 组件 | 用途 |
|------|------|
| `ScriptSelector` | 顶部剧本选择，下拉/标签 |
| `RoleSidebar` | 左侧角色列表，头像+名称+状态 |
| `RoleChatPanel` | 右侧会话区，消息列表+输入框 |
| `ProgressBar` | 处理进度条 |
| `ConfirmDispatchBar` | 确认/分发操作栏 |
| `OutputPreview` | 文本/图片/视频预览 |

### 3.4 响应式

- **桌面**：三块式完整展示
- **平板**：左侧可收折为图标栏
- **移动**：考虑底部 Tab 切换「剧本」「角色」「会话」

## 四、数据流与状态

### 4.1 核心实体

```
Project（项目）
  ├── scripts[]        # 剧本列表（多剧本并行）
  ├── materials[]      # 素材包
  ├── characters[]     # 自定义角色
  ├── config           # 剧本量、拼接字数、匹配章节
  └── pipelines[]      # 流水线任务

Pipeline（流水线任务）
  ├── projectId
  ├── scriptId         # 当前使用的剧本
  ├── steps[]          # 10 角色步骤
  └── status

Script（剧本）
  ├── source          # upload | outline_generated
  ├── content         # 全文
  ├── slices[]        # 切片结果（交给分镜）
  └── knowledgeBase   # 是否纳入知识库
```

### 4.2 多剧本并行

- **项目** 下可有多个 **剧本**，每个剧本可单独发起流水线
- **任务列表** 展示所有项目的流水线任务，支持按剧本筛选
- **工作台** 展示「当前选中项目」下的剧本与流水线

## 五、前端技术建议

| 模块 | 建议 | 说明 |
|------|------|------|
| 路由 | 保持 Vue Router | 新增 `/projects`、`/projects/:id`、`/knowledge` |
| 状态 | Pinia | 项目、当前剧本、流水线步骤 |
| 上传 | 原生 `<input type="file">` + FormData | zip/rar/图片，支持多选、拖拽 |
| 表单 | 保持现有 Tailwind | 配置项用折叠面板、输入框 |
| 列表 | 虚拟滚动（可选） | 剧本/任务多时性能优化 |

## 六、与现有实现的衔接

- **Sidebar**：保持现有（工作台、AI 对话、流水线、任务列表、系统设置）
- **路由**：`/pipelines` 为流水线页面，该页面内渲染三块式布局
- **Pipelines.vue**：由占位页改为三块式布局（剧本选择 + 角色列表 + 角色会话）
- **Chat**：右侧角色会话区复用现有 Chat 组件样式
- **主题**：沿用 `useTheme`、`--dp-*` 变量
