## 一、通用设计稿 Prompt（可直接复制）

```
请为「DramaPipeline 影视流水线」Web 控制台设计 UI 设计稿。

【项目定位】
工业级短剧生成工具，面向专业级用户（短剧创作者、导演、制片人）和企业用户（MCN、影视公司）。非全自动流水线，每步需用户确认后手动分发到下一阶段。

【整体风格】
- 控制台风格：专业、高效、可观测，偏工业感
- 布局：左侧固定导航栏 + 右侧主内容区
- 配色：深色主题，主色可选用 electric blue 或影视行业常用色（如深蓝、靛青、琥珀高亮）
- 视觉：玻璃拟态卡片（glass-card）、清晰的信息层级、状态标签醒目

【导航结构】（左侧 Sidebar）
1. 工作台 / 总览（Dashboard）
2. 流水线 / 任务列表（Pipeline Tasks）
3. 任务详情（Task Detail，含步骤确认与分发）
4. 角色管理 / 配置（Role Config）
5. 通知中心（Notifications，生成完成提醒）
6. 系统设置（Settings）

【核心页面与组件】
1. 工作台：项目概览、进行中任务数、待确认步骤数、快捷入口
2. 任务列表：表格或卡片列表，展示任务名称、当前步骤、状态、创建时间、操作
3. 任务详情：流水线步骤卡片（10 个角色）、每步含：角色名、输出预览区（文本/图片/视频）、确认按钮、分发按钮、状态标签（待执行/执行中/待确认/完成/失败）、可选回退/重做
4. 通知中心：消息列表，生成完成提醒、可标记已读
5. 步骤状态：待执行、执行中、待确认、完成、失败，需有明确视觉区分

【交互要点】
- 确认按钮：用户确认当前步骤输出
- 分发按钮：分发到下一阶段，触发下一步
- 进度条：展示流水线整体进度
- 支持多项目/多任务并行展示

【技术参考】
- 参考 zeroclaw 控制台布局与深色主题
- 前端技术栈：React + Tailwind
- 响应式：支持 1280px 及以上桌面端为主
```

## 二、分页面设计 Prompt

### 2.1 工作台 / Dashboard

```
设计 DramaPipeline 工作台页面：
- 左侧固定导航栏（深色），品牌名 DramaPipeline
- 主内容区：深色背景，顶部为「工作台」标题
- 核心数据卡片：进行中任务数、待确认步骤数、今日完成数
- 快捷入口：新建项目、查看任务列表
- 最近任务列表：展示最近 5～10 条任务，含名称、当前步骤、状态
- 风格：控制台、工业感、glass-card 卡片
```

### 2.2 任务列表页

```
设计 DramaPipeline 任务列表页面：
- 左侧导航 + 主内容区
- 顶部：标题「流水线任务」、筛选（状态、时间）、新建任务按钮
- 任务列表：表格形式，列包括：任务名称、项目、当前步骤、状态、创建时间、操作（查看/删除）
- 状态标签：待执行、执行中、待确认、完成、失败，颜色区分
- 支持分页或无限滚动
- 深色主题，专业控制台风格
```

### 2.3 任务详情页（核心）

```
设计 DramaPipeline 任务详情页（流水线步骤确认与分发）：
- 顶部：任务名称、整体进度条、返回按钮
- 主体：10 个流水线步骤卡片，垂直排列
  步骤顺序：剧本分析 → AI 导演 → 素材分析 → 分镜设计 → 首帧生成 → 视频生成 → 后期合成 → 角色创建 → 场景生成 → 审片
- 每个步骤卡片包含：
  - 步骤序号与角色名称
  - 状态标签（待执行/执行中/待确认/完成/失败）
  - 输出预览区：文本块、图片缩略图或视频播放器
  - 操作按钮：确认、分发到下一阶段、（可选）回退、重做
- 当前待确认步骤需高亮
- 已完成步骤可折叠
- 深色主题，控制台风格
```

### 2.4 通知中心

```
设计 DramaPipeline 通知中心页面：
- 左侧导航 + 主内容区
- 标题「通知中心」
- 消息列表：每条含标题、内容摘要、时间、已读/未读状态
- 类型：生成完成、任务失败、系统通知
- 支持全部已读、单条已读
- 深色主题
```

### 2.5 角色管理 / 配置

```
设计 DramaPipeline 角色配置页面：
- 展示 10 个流水线角色：剧本分析、AI 导演、素材分析、分镜设计、首帧生成、视频生成、后期合成、角色创建、场景生成、审片
- 每个角色可配置：启用/禁用、执行顺序、超时时间等
- 表格或卡片形式
- 深色控制台风格
```

## 三、组件级 Prompt（可复用）

### 步骤卡片

```
设计流水线步骤卡片组件：
- 卡片容器：圆角、玻璃拟态、边框
- 左侧：步骤序号、角色名称、状态标签
- 中间：输出预览区（支持文本、图片、视频）
- 右侧：确认按钮、分发按钮
- 状态颜色：待执行-灰、执行中-蓝、待确认-黄/琥珀、完成-绿、失败-红
```

### 状态标签

```
设计 5 种状态标签：待执行、执行中、待确认、完成、失败
- 小尺寸徽章样式
- 颜色区分，深色背景下清晰可读
- 可选图标辅助
```

### 侧边导航

```
设计控制台侧边导航：
- 固定宽度约 220px
- 深色背景，品牌 Logo + DramaPipeline 文字
- 导航项：工作台、流水线、任务、通知、配置
- 当前项高亮
- 底部可放系统设置
```

## 四、配色与规范建议

| 用途     | 建议值                         |
| -------- | ------------------------------ |
| 背景主色 | #0f1419 / #1a1f26（深灰黑）    |
| 卡片背景 | rgba 半透明 + 边框（glass）     |
| 主色     | #3b82f6（electric blue）或 #6366f1 |
| 成功     | #22c55e                        |
| 警告     | #f59e0b                        |
| 失败     | #ef4444                        |
| 文字主色 | #e5e7eb                        |
| 文字次要 | #9ca3af                        |

## 五、使用说明

1. **Figma**：将「一、通用设计稿 Prompt」或「二、分页面」内容粘贴到 Figma AI 或插件中生成初稿
2. **设计师**：作为设计需求文档，交付给设计师按规范出图
3. **迭代**：可根据实际开发反馈调整配色、布局细节后重新生成
